debedder :: toy

local no-vectordb semantic decoder — plain fixed-vocab linear layer + bigram constrained decode. all math client-side, no API calls.
text → codepoint histogram (26-d) → this stands in for a real embedder (bge/e5/etc). same idea as edgeEmbed, just fewer dims for visibility.
word → dictionary.indexOf(word) → slot index. real table, built once from corpus. unknown words have no slot at all — that's the tradeoff vs hashing.
corpus (24 toy sentences) → tf-ish top keywords per sentence → vocab index → multi-label targets.
train W [vocab-size × 26 dims] via logistic regression / SGD, live in this tab. no embedder in the training loop — that's the whole simplification.

type any sentence (even unseen ones) → toy-embed → W → top-k vocab slots → candidate words → bigram model strings them into text, biased toward hitting the predicted keywords.

try sentences totally outside the toy corpus — words not in the fixed vocab table have no slot at all (see panel 02), so out-of-vocab meaning gets dropped entirely rather than smeared into a nearby bucket. the bigram model still falls back gracefully on whatever keywords do land. this is the tradeoff for dropping the hash trick: cleaner slots, harder ceiling on coverage.
